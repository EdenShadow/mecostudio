# Kimi Search 🔍

使用 **Kimi AI** 进行联网搜索，自动整理搜索结果并构建本地知识库。支持生成索引和快捷说明书，是创建 Agents、查找人设信息、构建知识库的利器。

## 核心功能

- 🌐 **联网搜索** - 使用 Kimi AI 实时搜索互联网最新信息
- 📚 **知识库构建** - 自动整理搜索结果并保存到本地
- 📑 **智能索引** - 自动生成知识库索引，方便查阅
- 📖 **快捷指南** - 自动生成使用说明书和最佳实践
- 🗂️ **分类管理** - 支持 agents/tech/research/reference 等分类
- ⚡ **多种深度** - 支持快速/标准/深度三种搜索模式

## 快速开始

```bash
# 基础搜索
kimi-search "搜索内容"

# 指定分类搜索
kimi-search "赛博朋克AI助手性格特点" --category agents

# 深度搜索
kimi-search "React Server Components 最佳实践" --depth deep

# 更新索引
kimi-search index
```

## 命令详解

### 1. 基础搜索

```bash
kimi-search <查询内容> [选项]
```

**选项：**
- `--category, -c <name>` - 分类目录（默认: general）
  - `agents` - Agents人设信息
  - `tech` - 技术资料
  - `research` - 研究报告
  - `reference` - 参考资料
  - `general` - 通用搜索
  
- `--depth, -d <level>` - 搜索深度
  - `quick` - 快速搜索 (~500字)
  - `standard` - 标准搜索 (~1000-1500字)
  - `deep` - 深度搜索 (~2000-3000字)

- `--context, -x <text>` - 添加上下文背景

### 2. 知识库管理

```bash
# 更新知识库索引
kimi-search index

# 生成快捷使用指南
kimi-search guide

# 列出知识库内容
kimi-search list
```

## 使用场景

### 场景1: 创建新Agent时查找人设

```bash
kimi-search "赛博朋克风格AI助手的性格特点和说话方式" \
  --category agents \
  --context "用于RPG游戏中的黑客角色"
```

**生成的资料包括：**
- 角色性格特征
- 说话方式和语气
- 常用词汇和表达
- 背景故事元素

### 场景2: 寻找技术知识库

```bash
kimi-search "MCP (Model Context Protocol) 协议规范和实现" \
  --category tech \
  --depth deep
```

**生成的资料包括：**
- 协议规范说明
- 实现示例
- 最佳实践
- 相关工具和资源

### 场景3: 快速事实查证

```bash
kimi-search "OpenAI o3-mini 模型特点和发布时间" --depth quick
```

### 场景4: 构建领域知识库

```bash
kimi-search "2024年人工智能行业发展趋势、市场规模、主要玩家" \
  --category research \
  --depth deep
```

## 依赖要求

- **kimi-cli** - 必须安装并配置好API Key
- **Node.js** >= 18.0.0

安装 kimi-cli：
```bash
# macOS/Linux
curl -LsSf https://code.kimi.com/install.sh | bash

# 或使用 uv
uv tool install --python 3.13 kimi-cli
```

配置 API Key：
```bash
kimi
# 在交互界面输入 /login 配置 Kimi API Key
```

## 搜索技巧

### 1. 精准关键词

```bash
# 好的查询
kimi-search "Python asyncio 并发最佳实践 性能优化"

# 避免太宽泛
kimi-search "Python编程"  # ❌ 太宽泛
```

### 2. 使用上下文

```bash
kimi-search "角色扮演游戏NPC" \
  --category agents \
  --context "中世纪奇幻背景，需要一个商人NPC"
```

### 3. 选择合适的深度

- **`quick`** - 快速查证事实、获取概要
- **`standard`** - 一般研究、收集资料
- **`deep`** - 深度研究、构建完整知识库

### 4. 定期整理

```bash
# 每周更新一次索引
kimi-search index

# 定期查看知识库
kimi-search list
```

## 故障排除

### 错误：kimi command not found

```bash
# 检查 kimi 是否安装
which kimi

# 重新安装
uv tool install kimi-cli
```

### 错误：API Key 未配置

```bash
# 启动 kimi 配置
kimi
# 输入 /login 配置 API Key
```

### 错误：搜索超时

- 深度搜索可能需要 2-3 分钟
- 检查网络连接
- 尝试使用 `--depth quick` 进行快速搜索

## 最佳实践

1. **分类管理** - 根据用途选择合适的分类
2. **定期索引** - 使用 `kimi-search index` 保持索引更新
3. **版本控制** - 将知识库纳入版本控制（可选）
4. **清理旧资料** - 定期清理过时信息
5. **组合搜索** - 先quick查证，再deep深入研究
